Linux Practice - Day 6

Posted on 2024-12-01 In Linux

Question:

You are managing a Linux server and need to perform a few common system administration tasks:

  1. Create a new user named deploy with a home directory /home/deploy and set the shell to /bin/bash.
  2. Set a password for the deploy user and ensure the password is secure (at least 8 characters with a mix of uppercase, lowercase, and special characters).
  3. Assign the deploy user to the sudo group to allow them to run administrative commands with sudo.
  4. Configure the deploy user to have restricted access:
    • Set up a chroot environment, meaning the user should only have access to their home directory.
    • Ensure the user cannot access system directories such as /etc, /bin, etc.
  5. Set up a cron job for the deploy user to run a script /home/deploy/backup.sh every day at 2:00 AM.
  6. Check if the deploy user can successfully run sudo and run a simple command like sudo ls /root.

Hints:

  • Use useradd and passwd for user creation and password setting.
  • Use usermod to add the user to the sudo group.
  • Use chroot to restrict access for the user.
  • Use crontab for scheduling tasks.
  • Use sudo to test elevated privileges.

Linux Practice - Day 5

Posted on 2024-11-30 In Linux

Question:

You are tasked with maintaining a Linux server and performing regular system administration tasks. Your goal is to:

  1. List all users on the system who have not logged in for the past 30 days and save the list to /home/yourusername/inactive_users.txt.

  2. Check the disk space of the /home directory, and if the disk usage exceeds 90%, send a warning email to admin@yourdomain.com.

  3. Set up log rotation for the /var/log/myapp.log file:

    • Rotate the log file every week.
    • Keep 4 weeks’ worth of logs and delete older logs.
    • Ensure that the log file is compressed after rotation.

  4. Create a script that:

    • Checks the available disk space of the /home directory.
    • Runs the disk space check every hour, and sends a notification email if the usage exceeds 90%.

Hints:

  • Use last or lastlog to check user logins.
  • Use df to check disk space usage.
  • Use mail or sendmail to send email notifications.
  • Use logrotate for managing log rotation.
  • Use cron to schedule tasks.

Linux Practice - Day 4

Posted on 2024-11-29 In Linux

Question:

You are tasked with managing disk space and backups on a Linux server. Perform the following tasks:

  1. Identify the disk usage of all mounted filesystems and save the output to /home/yourusername/disk_usage_report.txt.
  2. Find all files larger than 100MB in the /var directory and its subdirectories and save the list (with file sizes) to /home/yourusername/large_files.txt.
  3. Create a compressed backup of the /etc directory named etc_backup.tar.gz and store it in /home/yourusername/backups/. Ensure the backup includes hidden files.

Hints:

  • Use df to check disk usage.

Linux Practice - Day 3

Question:

You are managing system performance on a Linux server. Your task is to:

  1. Monitor real-time system resource usage (CPU, memory, disk I/O) for one minute and save the output to a file named system_stats.log in your home directory.

  1. Create a new user called developer with a home directory /home/developer.
  2. Set a password for the developer user and make sure it expires in 30 days.
  3. Create a group called devteam and add the developer user to it.
  4. Set up a shared directory /projects that is accessible to the devteam group.
  • The directory should have the following permissions:
  • The devteam group should have read, write, and execute permissions.
  • Others should only have read and execute permissions.

  1. Verify the permissions on the /projects directory to ensure they are set correctly.

Hints:
