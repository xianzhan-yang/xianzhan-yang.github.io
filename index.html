Linux Practice - Day 2

Posted on 2024-11-27 In Linux

Question:

You are working on a Linux system and need to manage users and permissions. Your task is to:

  1. Create a new user called developer with a home directory /home/developer.
  2. Set a password for the developer user and make sure it expires in 30 days.
  3. Create a group called devteam and add the developer user to it.
  4. Set up a shared directory /projects that is accessible to the devteam group.
  • The directory should have the following permissions:
  • The devteam group should have read, write, and execute permissions.
  • Others should only have read and execute permissions.
  • New files in the directory should inherit the group permissions.
  1. Verify the permissions on the /projects directory to ensure they are set correctly.

Hints:

  • Use useradd, passwd, groupadd, and usermod to manage users and groups.
  • Use chmod, chown, and setfacl for setting up permissions.
  • Use chgrp to change the group ownership of a directory.

Question:

You are tasked with managing a web server running on a Linux system. The server log files are stored in /var/log/apache2/. The logs are consuming too much disk space. Your task is to:

  1. Find the five largest log files in the directory and display their sizes in human-readable format.
  2. Archive the log files older than 30 days into a compressed file named old_logs.tar.gz in your home directory.
  3. Set up a cron job to delete log files older than 30 days automatically every week.

Hints:

  • Use du, ls, or find for file size and date filtering.
  • Use tar for archiving.
  • Use crontab to set up the cron job.
