Posted on 2024-12-01 In Linux

Question:

You are managing a Linux server and need to perform a few common system administration tasks:

  1. Create a new user named deploy with a home directory /home/deploy and set the shell to /bin/bash.
  2. Set a password for the deploy user and ensure the password is secure (at least 8 characters with a mix of uppercase, lowercase, and special characters).
  3. Assign the deploy user to the sudo group to allow them to run administrative commands with sudo.
  4. Configure the deploy user to have restricted access:
    • Set up a chroot environment, meaning the user should only have access to their home directory.
    • Ensure the user cannot access system directories such as /etc, /bin, etc.
  5. Set up a cron job for the deploy user to run a script /home/deploy/backup.sh every day at 2:00 AM.
  6. Check if the deploy user can successfully run sudo and run a simple command like sudo ls /root.

Hints:

  • Use useradd and passwd for user creation and password setting.
  • Use usermod to add the user to the sudo group.
  • Use chroot to restrict access for the user.
  • Use crontab for scheduling tasks.
  • Use sudo to test elevated privileges.
