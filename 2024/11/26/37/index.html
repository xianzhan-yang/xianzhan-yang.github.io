Linux Practice - Day 1

Posted on 2024-11-26 Edited on 2024-11-27 In Linux

Question:

You are tasked with managing a web server running on a Linux system. The server log files are stored in /var/log/apache2/. The logs are consuming too much disk space. Your task is to:

  1. Find the five largest log files in the directory and display their sizes in human-readable format.
  2. Archive the log files older than 30 days into a compressed file named old_logs.tar.gz in your home directory.
  3. Set up a cron job to delete log files older than 30 days automatically every week.

Hints:

  • Use du, ls, or find for file size and date filtering.
  • Use tar for archiving.
  • Use crontab to set up the cron job.

Answer:

Step 1: Find the five largest log files in the directory and display their sizes in human-readable format.

Use the du command to check the sizes of files and the sort command to display the largest ones:

Explanation:

  • du -ah /var/log/apache2/ lists the sizes of all files and directories under /var/log/apache2/.
  • sort -rh sorts the output by human-readable size in reverse order (largest first).
  • head -n 5 displays only the top 5 results.

Step 2: Archive the log files older than 30 days into a compressed file named old_logs.tar.gz in your home directory.

Use the find command to identify files older than 30 days and then pipe the results into tar to create the archive.

Explanation:

  • find /var/log/apache2/ -type f -mtime +30 finds all files (-type f) under /var/log/apache2/ that are older than 30 days (-mtime +30).
  • -exec tar -czf /old_logs.tar.gz {} + uses tar to compress these files into a tar.gz archive named old_logs.tar.gz in your home directory (/).
